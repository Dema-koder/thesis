\begin{abstract}

This thesis evaluates the performance of six Java garbage collector
configurations --- Serial GC, Parallel GC, G1 GC, ZGC, Generational ZGC,
and Shenandoah GC --- on a representative Spring Boot web application under
four workload scenarios and three heap sizes, using JDK~21~LTS. Existing
research relies on synthetic benchmarks (DaCapo) or specialized systems
(Apache Cassandra), leaving a gap in understanding GC behavior in common
web services with REST APIs, ORM persistence, and application-level caching.

The study measures tail latency percentiles (p95, p99, p999), throughput,
and GC pause characteristics across read-heavy, write-heavy, high-concurrency,
and burst traffic scenarios. Key findings include: no single collector is
universally optimal --- Parallel GC achieves the best write-heavy p99
(6\,ms), ZGC leads under read-heavy load (p99 11\,ms, max pause 0.05\,ms),
and Shenandoah provides uniquely stable latency under burst traffic
($\Delta$p99\,$=$\,1\,ms across load phases). Generational ZGC matches
Parallel GC's write-heavy performance while maintaining sub-millisecond
GC pauses.

Results are synthesized into a GC selection decision tree, per-collector
JVM configuration templates, and production migration playbooks for
Spring Boot deployments on JDK~21.

\textbf{Keywords}: Java, garbage collection, JVM, G1 GC, ZGC, Shenandoah,
Generational ZGC, performance benchmarking, Spring Boot, tail latency.

\end{abstract}
